# AI Prompt Engineering 技巧与实践

## 什么是 Prompt Engineering？

Prompt Engineering 是一门设计和优化提示词的艺术，目的是让 AI 模型生成更准确、更有用的回复。

## 基础原则

### 1. 清晰明确

```
❌ 不好的例子：
"写个东西"

✅ 好的例子：
"写一篇关于 Python 列表推导式的技术博客文章，
目标读者是中级开发者，包含代码示例和最佳实践"
```

### 2. 提供上下文

```
❌ 不好的例子：
"怎么做蛋糕？"

✅ 好的例子：
"我想做一个巧克力蛋糕，家里有以下材料：
- 面粉 200g
- 鸡蛋 3个
- 巧克力 100g
- 牛奶 100ml
请给我一个详细的制作步骤"
```

### 3. 指定格式

```
❌ 不好的例子：
"推荐一些电影"

✅ 好的例子：
"推荐5部科幻电影，以JSON格式返回，包含以下字段：
- title: 电影名
- year: 年份
- genre: 类型
- rating: 评分（1-10）
- summary: 简短简介"
```

## 核心技巧

### 技巧1：角色设定 (Role Prompting)

```
你是一位资深的前端架构师，有10年以上的Vue开发经验。
请从技术可行性、性能优化、代码可维护性三个角度
评估以下代码，并给出具体的改进建议。

[代码...]
```

### 技巧2：思维链 (Chain of Thought)

```
❌ 不好的例子：
"24 + 37 * 2 = ?"

✅ 好的例子：
"请先思考解决这个问题的步骤，然后给出答案：
24 + 37 * 2 = ?

思考步骤：
1. 先计算乘法：37 * 2 = 74
2. 再计算加法：24 + 74 = 98

答案：98"
```

### 技巧3：少样本学习 (Few-shot Learning)

```
请根据以下示例，将英文翻译成中文：

示例：
Hello -> 你好
How are you? -> 你好吗？
Thank you -> 谢谢

现在翻译：
Good morning ->
```

### 技巧4：结构化输出

```
请分析以下用户反馈，并按以下格式返回：

{
  "sentiment": "positive|negative|neutral",
  "key_points": ["点1", "点2"],
  "suggestions": ["建议1", "建议2"],
  "priority": "high|medium|low"
}

用户反馈：
"这个软件功能很强大，但是界面太复杂了，
新手很难上手。希望能增加新手引导教程。"
```

## 常用 Prompt 模板

### 模板1：代码审查

```
作为代码审查专家，请审查以下代码：

[代码...]

请从以下方面审查：
1. 代码质量
2. 安全性
3. 性能
4. 可维护性

对每个问题，请提供：
- 问题描述
- 严重程度（高/中/低）
- 修改建议
- 修改后的代码示例
```

### 模板2：文章写作

```
请帮我写一篇技术博客文章，要求如下：

主题：[主题]
目标读者：[初级/中级/高级]
文章结构：
1. 引言（为什么需要这个技术）
2. 基本概念介绍
3. 实践示例（含代码）
4. 最佳实践
5. 总结

要求：
- 语言通俗易懂
- 代码示例完整可运行
- 提供实用建议
- 字数约2000字
```

### 模板3：问题解决

```
我遇到了一个问题，请帮我分析和解决：

问题描述：
[详细描述你的问题]

我的环境：
- 操作系统：
- 软件版本：
- 相关配置：

我已尝试的解决方案：
1. [尝试1]
2. [尝试2]

请提供：
1. 问题原因分析
2. 具体的解决步骤
3. 预防措施
```

### 模板4：学习计划

```
我想学习[技能]，请帮我制定学习计划：

我的情况：
- 当前水平：[零基础/有基础/熟练]
- 每天可用时间：[X小时]
- 学习目标：[具体目标]

请制定一个[4周/8周/12周]的学习计划，包含：
- 每周学习主题
- 推荐的学习资源
- 实践项目
- 检验标准
```

## 高级技巧

### 技巧1：自我一致性 (Self-Consistency)

```
请用3种不同的方式回答这个问题，然后选择最合理的答案：

[问题...]

方式1：
[回答1]

方式2：
[回答2]

方式3：
[回答3]

综合分析后，最终答案：
```

### 技巧2：迭代优化

```
这是我写的代码，请帮我逐步优化：

[原始代码...]

第1轮优化（功能完整）：
[优化后的代码]

第2轮优化（性能提升）：
[优化后的代码]

第3轮优化（代码规范）：
[优化后的代码]

最终版本：
[最终代码]

优化总结：
```

### 技巧3：多专家辩论

```
请模拟3位专家讨论这个问题：

问题：[问题...]

专家A（保守派）：
[观点A]

专家B（激进派）：
[观点B]

专家C（中立派）：
[观点C]

综合各方观点，最终建议：
```

## 常见错误与避免

### ❌ 错误1：过于简短

```
"写代码"

✅ 改为：
"使用 Python 的 requests 库写一个获取
GitHub 仓库信息的函数，包含错误处理和重试逻辑"
```

### ❌ 错误2：过于模糊

```
"做得更好一点"

✅ 改为：
"优化这段代码的性能，目标是将执行时间
从500ms降低到100ms以内"
```

### ❌ 错误3：缺少约束

```
"推荐一本书"

✅ 改为：
"推荐一本关于机器学习的入门书籍，
要求：
- 作者知名
- 有实际项目
- 2020年后出版
- 适合零基础"
```

## 最佳实践总结

1. **清晰具体**：明确说明你想要什么
2. **提供上下文**：给出必要的背景信息
3. **指定格式**：告诉AI如何组织输出
4. **给示例**：用few-shot让AI理解期望
5. **要求思考**：用CoT让AI展示推理过程
6. **迭代优化**：逐步调整和改进
7. **验证结果**：检查AI的输出是否正确
